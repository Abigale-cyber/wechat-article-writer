# 创作流程与框架

来源：教案 04《创作逻辑》

## 母规则

**创作是对话，不是单向输出。** 写每一个字之前，想清楚对面坐的是谁，他在想什么，他为什么要听你说下去。

**没有观点的内容是信息搬运。** 有观点才是创作。如果 brief 的核心判断站不住，先回去改 brief，不要硬写。

## 三种选题方法

### 方法一：痛点挖掘法（从读者问题出发）
- 目标用户在关心什么？有什么问题没被解决？
- 评论区、社群、咨询记录都是选题金矿

### 方法二：热点借势法（从时事切入）
- 关注微博热搜、抖音热点、公众号爆文
- 速度是关键，当天蹭效果最好
- 把热点和你的定位做关联

### 方法三：经验提炼法（从个人经历提炼）
- 拆解对标账号的爆款内容
- 分析选题角度、标题、开头
- 用自己的观点和案例重新演绎

## 七步写作流程

| 步骤 | 做什么 | 要点 |
|------|--------|------|
| 1. 选题 | 从选题库选一个今天要写的主题 | 好选题决定一半以上流量 |
| 2. 观点 | 对选题形成自己的立场 | 没有观点 = 信息搬运 |
| 3. 框架 | 根据内容类型选框架 | 见下方五大框架 |
| 4. 开头 | 写开头（见 hook-and-ending.md） | 前 3 行定生死 |
| 5. 正文 | 填充框架，用案例/数据/故事支撑 | 每段不超过 5 行 |
| 6. 结尾 | 推动互动或行动（见 hook-and-ending.md） | 转发/评论/行动号召 |
| 7. 标题 | 反复打磨标题（见 title-and-packaging.md） | 标题是流量入口 |

## 五大行文框架

### 框架一：故事类
- **适用：** 传递价值观、建立人设
- **结构：** 事件 → 冲突 → 转折 → 感悟
- **重点：** 要有真实感和画面感

### 框架二：观点类
- **适用：** 输出立场、引发讨论
- **结构：** 观点 → 论据（案例/数据） → 总结
- **重点：** 观点要鲜明，论据要扎实

### 框架三：热点类
- **适用：** 蹭流量、借势传播
- **结构：** 热点事件 → 关联分析 → 你的观点
- **重点：** 速度要快、角度要独特

### 框架四：清单类
- **适用：** 提供实用价值、干货分享
- **结构：** 总起 → 分点罗列 → 总结
- **重点：** 每点要简洁、可直接执行

### 框架五：教学类
- **适用：** 展示专业能力、建立权威
- **结构：** 问题 → 方法 → 步骤 → 注意事项
- **重点：** 步骤要具体、有可操作性

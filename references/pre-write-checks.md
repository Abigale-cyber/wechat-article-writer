# 写前门控与写后自检

## 写前门控（开始写作之前必须通过）

拿到 brief 后，先检查这 3 项再动笔。任何一项不通过，先回去改 brief 或跟用户确认：

### 1. 核心判断够硬吗？

- 能不能一句话说清楚"这篇文章到底想说什么"？
- 如果听起来像维基百科条目 → 不合格
- 如果"从多个角度分析" → 不合格（必须有立场）
- 如果读者看完只会说"哦，知道了" → 不够硬

### 2. 读者够具体吗？

- 能不能描述出一个具体的读者场景？
- "所有人" → 不合格
- "对 AI 感兴趣的人" → 还是太泛
- "用 AI 写了一个月公众号但产出不稳定的人" → 合格

### 3. 我有足够的话可说吗？

- brief 中至少有 3 条可用素材
- 每个章节有明确的观点和支撑
- 如果素材不够 → 回去补素材，不硬写

## 写后自检清单

全文写完后，逐条过一遍再交给用户：

### 标题
- [ ] 是否遵循 5 大原则之一
- [ ] 是否在 15 字以内
- [ ] 3 个候选中是否至少有 1 个能引发好奇心

### 开头
- [ ] 前 3 行是否足够吸引人
- [ ] 是否用了 8 种模板之一
- [ ] 是否在 3-4 句内点明了文章要解决的问题

### 正文
- [ ] 是否有明确观点（不是信息搬运）
- [ ] 逻辑是否通顺（每段首句是核心观点）
- [ ] 是否有案例/数据支撑
- [ ] 每段是否不超过 5 行
- [ ] 字数是否在 1500-3000 字之间

### 结尾
- [ ] 是否推动了互动或行动
- [ ] 是否有行动号召

### 排版
- [ ] 加粗是否只用一种颜色
- [ ] 是否两端对齐
- [ ] 段落间距是否舒适

### AI 痕迹
- [ ] 读一遍，有没有明显的 AI 生成痕迹
- [ ] 句子长度是否有变化（不是每句都差不多长）
- [ ] 有没有个人化的表达（不是纯客观陈述）

# 框架选择与写作规则

## 母规则

创作是对话。写每一个字之前，想清楚对面坐的是谁，他在想什么，他为什么要听你说下去。

## 五大框架选择指南

### 故事类
- **适用：** 传递价值观、建立人设
- **结构：** 事件 → 冲突 → 转折 → 感悟
- **示例：** "我用 AI 写了一个月公众号，数据是这样的"

### 观点类
- **适用：** 输出立场、引发讨论
- **结构：** 观点 → 论据 → 总结
- **示例：** "AI 不会取代创作者，但会淘汰内容搬运工"

### 热点类
- **适用：** 蹭热点、带观点
- **结构：** 热点引入 → 分析 → 关联自身
- **示例：** "ChatGPT 又更新了，但对写公众号的人来说只有一件事值得注意"

### 清单类
- **适用：** 干货分享、步骤教学
- **结构：** 总述 → 分点列举 → 总结
- **示例：** "写公众号 3 个月，这 5 个坑我都踩过"

### 教学类
- **适用：** 教程、方法论
- **结构：** 问题 → 方法 → 步骤 → 案例 → 总结
- **示例：** "如何用 AI 在 30 分钟内写出一篇合格的公众号文章"

## 分段写作法

每个章节独立写作，不要一口气写完全文：

1. **先写首句**：这一段的核心观点，放在第一句
2. **证明它**：用案例、数据、对比来支撑
3. **给一个可操作的 takeaway**：读者能拿走什么
4. **过渡到下一段**

## 开头原则

开头只有一个任务：让读者相信这篇文章会回答他的问题。3-4 句话到位。

公式：
1. 点出读者面临的问题或处境
2. 告诉他这篇文章要做什么
3. 可选：给他一个相信你的理由

避免：
- "在当今数字化时代..."
- 用问题开头（除非问题真的很犀利）
- 前三句全是背景铺垫

## 正文原则

- 每段不超过 5 行
- 每句话不超过 25 字
- 用"我们"、"你"拉近距离
- 具体案例 > 抽象描述
- 先说观点，再说理由
- 每个章节独立交付价值（读者跳着看也能 get 到东西）

## 结尾原则

三个要素：
1. 核心观点的 1-2 句总结
2. 告诉读者下一步该做什么
3. 如果适合，推动互动（评论/转发）

## 文章类型预设

预设 = 框架 × 语调 × 配图风格 × 结尾模板 × 排版倾向的预组合。选定预设后，写作时自动对齐所有维度。

| 预设 | 框架 | 语调 | 配图风格 | 结尾模板 | 适用 |
|------|------|------|---------|---------|------|
| `deep-insight` | 观点类 | 冷静专业，克制有分量 | blueprint（技术蓝图） | 行动号召 | 行业分析、深度思考 |
| `story-time` | 故事类 | 温暖亲切，像朋友聊天 | warm watercolor（柔和水彩） | 金句收束 | 个人经历、成长故事 |
| `hot-take` | 热点类 | 锋利直接，节奏快 | bold graphic（高对比） | 互动引导（提问/投票） | 时事评论、热点解读 |
| `how-to` | 教学类 | 清晰耐心，步骤感强 | clean infographic（极简信息图） | 步骤总结 + 行动号召 | 教程、方法论、工具使用 |
| `quick-list` | 清单类 | 轻松直给，信息密度高 | notion（极简线画） | 一个核心总结 | 干货合集、踩坑总结 |

**使用方式：**
- brief 中如果指定了推荐框架，自动匹配对应预设
- 用户可覆盖：直接说"用 story-time 预设写"
- 扩展：通过 EXTEND.md 追加自定义预设（见 `_shared/extend-system.md`）

## 去 AI 痕迹

写完全文后，做一遍反 AI 检查。以下模式必须消除：

| AI 痕迹 | 修复方式 |
|---------|---------|
| "在当今/在这个XX时代" | 删掉，直接进入正题 |
| 三连排比（A、B、C） | 只保留最强的两个，或者换成不规则的节奏 |
| "不仅...而且...更..." | 拆成独立的短句 |
| 破折号过多 | 用句号替代 |
| "值得注意的是"/"需要指出的是" | 直接说结论 |
| 升华式结尾（"让我们一起..."） | 换成具体的行动号召 |
| "综合来看"/"总而言之" | 直接给结论 |
| 每段长度一样 | 刻意制造长短交替 |
| 所有句子都是陈述句 | 插入问句、感叹句打破节奏 |
